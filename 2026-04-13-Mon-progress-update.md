## How the Dead Reckoning Parameters Were Hardcoded

The heuristic dead reckoning algorithm is used to estimate where the toothbrush cursor should be placed on screen. All of its parameters were written directly into the source code as fixed numbers. These numbers are not changed or recalculated when the program is run.

The following parameters were hardcoded in the `estimate_dead_reckoning` function (in `core.py`, lines 333–355):

- A **velocity damping factor** of **0.84** was set. It is used to reduce the estimated velocity over time so that old motion does not affect the current position too much.
- An **acceleration scaling factor** of **0.10** was set. It is used to control how much the sensor's acceleration values change the estimated velocity.
- A **yaw contribution factor** of **0.0012** was set. It is used to shift the horizontal position when a left-right rotation is detected.
- A **pitch contribution factor** of **0.0015** was set. It is used to shift the vertical position when an up-down tilt is detected.

More parameters were hardcoded in the `analyze_session` function (in `core.py`, lines 516–524), where the dead reckoning result is combined with the zone prediction to move the cursor:

- A **scaling factor** of **0.38** was applied to both the horizontal and vertical dead reckoning values.
- The horizontal value was limited to a range of **-0.18 to 0.18**. The vertical value was limited to **-0.16 to 0.16**.
- A **cursor follow speed** was defined as **0.40 + (0.35 x activity level)**. This means the cursor is moved faster when stronger brushing activity is detected.

Because all of these values were written as fixed numbers in the code, the same parameters are used every time the program is run. No new configuration is needed before a video is generated.

## How to Generate a New Video

A new coverage video can be generated from any `.txt` sensor log file. The following steps should be followed.

### Step 1 — Check that the program is installed

It should be confirmed that the `ringbrush-coverage` package has been installed. If it has not been installed yet, the following command can be run from the project folder:

```bash
pip install -e .
```

### Step 2 — Run the command

The video is generated by running the following command:

```bash
python -m ringbrush_coverage path/to/logfile.txt -o path/to/output.mp4
```

- The path to the `.txt` sensor log file must be given as the first argument.
- The `-o` option can be used to choose where the output `.mp4` file is saved. If `-o` is not given, the video will be saved in the current folder with the name `<logfile-stem>-coverage.mp4`.

### Step 3 — Calibration (handled automatically)

No manual calibration step is needed. If labeled calibration files (for example, `outer-front-only.txt`, `inner-upper-only.txt`, `no-movement-idle.txt`, etc.) are found in the same folder as the input log file, they will be discovered and used automatically. If no calibration files are found, the program's built-in default calibration values will be used instead.

### Step 4 — Check the output

After the command is finished, a summary of the coverage results will be printed to the screen. The `.mp4` video file will be available at the chosen output path.

For example, the following command was used to produce the video for this session:

```bash
python -m ringbrush_coverage "C:/MSc-Computer-Science/Semester-2/pdss/2026-04-12_2127_up-and-down-and-up-and-down-and.txt" -o outputs/up-and-down-and-up-and-down-and.mp4
```

The calibration files in the `pdss` folder were detected automatically, and the output video was saved to the `outputs` folder.
