# Session — 2026-04-27: Render layout cleanup and switch to 1280x720

## Summary

The MP4 renders were upgraded from 640x360 to 1280x720, and the on-screen
layout was rebuilt around an explicit non-overlapping grid. The previous
layout placed each UI box (header text, mouth visualization, coverage
sidebar, status pill, timeline) using absolute pixel offsets; at 640x360
those offsets caused multiple boxes to occupy the same pixels, which
produced the messy, hard-to-read frame the user reported.

All four reference outputs were regenerated under the new code and
committed alongside it. Older 640x360 renders were renamed with the
`_3e85611` commit suffix and retained on disk so the two generations
remain distinguishable.

## What changed in the code

- [`ringbrush_coverage/render.py`](../ringbrush_coverage/render.py) — added
  `_layout(width, height)` that returns boxes for all UI regions; the boxes
  are guaranteed not to overlap and reserve margin for the mouth's drop
  shadow and lip arcs. `_build_static_base` and `_draw_frame` now read all
  geometry from this layout dict instead of recomputing absolute pixel
  offsets. Font sizes, decoration offsets, cursor radius, trail width,
  outline thickness, and corner radii all scale with the canvas size.
- [`ringbrush_coverage/cli.py`](../ringbrush_coverage/cli.py) — `--width`
  default changed from 640 to 1280; `--height` default changed from 360 to
  720. CLI flags still accept any size.
- [`README.md`](../README.md) — default-settings line updated.
- Coverage sidebar now displays a percentage readout next to each region
  label, in addition to the bar fill.

## Follow-ups for future sessions

- The layout function is resolution-agnostic, but it has only been
  visually verified at 1280x720. If a future session wants to render at
  a different aspect ratio (e.g. 1920x1080 or square), it would be worth
  re-running a single-frame visual check before committing a long render.
- The mouth polygon coordinates (`ZONE_POLYGONS` in `render.py`) are
  authored against a roughly 2.7:1 aspect ratio. The new mouth box at
  1280x720 is closer to 2.1:1, which makes the polygons read as slightly
  taller-than-wide pentagons. Acceptable but a future cosmetic pass could
  re-author the polygons against the new aspect.
